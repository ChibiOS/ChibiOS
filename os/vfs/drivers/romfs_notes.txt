- ROM FS exposes as VFS driver a const representation of a file system.
- A specific tool converts a files tree as a C const representation to be included in project.
- ROMFS constructor takes a pointer to the const data.
- ROM files can be compressed or uncompressed, a field in the file header represents this,
  potentially multiple compression types could be used.
- Support for dynamic files, content is generated by calling a callback with appropriate
  parameters.
- The representation contains a list of all directory in alphabetic order (full paths).
- No need to implement current directory in the driver.
- Each directory will be a list of file headers followed by content, it could be a linked list.
